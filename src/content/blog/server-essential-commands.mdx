# 服务器基础配置指南

本文记录了在配置服务器和 MongoDB 数据库过程中最常用的命令，主要包括系统管理、MongoDB 配置等核心操作。

## 1. 系统基础命令

```bash
# 系统信息查看
df -h                 # 查看磁盘使用情况
free -h               # 查看内存使用情况
top                   # 查看系统资源占用

# 文件操作
ls -la               # 查看文件详细信息
mkdir dirname        # 创建目录
rm -rf dirname      # 删除目录及其内容
```

## 2. MongoDB 安装与配置

```bash
# 安装 MongoDB
wget -qO - https://www.mongodb.org/static/pgp/server-7.0.asc | sudo apt-key add -
echo "deb [ arch=amd64,arm64 ] https://repo.mongodb.org/apt/ubuntu focal/mongodb-org/7.0 multiverse" | sudo tee /etc/apt/sources.list.d/mongodb-org-7.0.list
sudo apt update
sudo apt install -y mongodb-org

# 服务管理
sudo systemctl start mongod    # 启动 MongoDB
sudo systemctl status mongod   # 查看状态
sudo systemctl enable mongod   # 设置开机自启
sudo systemctl stop mongod     # 停止服务

# 查看日志
sudo tail -f /var/log/mongodb/mongod.log
```

## 3. MongoDB 用户管理

```bash
# 连接数据库
mongosh              # 进入 MongoDB Shell

# 创建管理员用户
use admin
db.createUser({
  user: "admin",
  pwd: "your_password",  // 替换为实际密码
  roles: [
    { role: "userAdminAnyDatabase", db: "admin" },
    { role: "readWriteAnyDatabase", db: "admin" }
  ]
})

# 创建应用数据库用户
use your_database    # 切换到你的应用数据库
db.createUser({
  user: "app_user",
  pwd: "your_password",  // 替换为实际密码
  roles: [
    { role: "readWrite", db: "your_database" }
  ]
})
```

## 4. MongoDB 安全配置

```bash
# 编辑 MongoDB 配置文件
sudo vim /etc/mongod.conf

# 重要配置项
security:
  authorization: enabled    # 启用身份验证

net:
  port: 27017
  bindIp: 0.0.0.0         # 允许远程连接（确保配置了防火墙）

# 重启服务使配置生效
sudo systemctl restart mongod
```

## 5. 防火墙配置

```bash
# UFW 防火墙管理
sudo ufw status                    # 查看防火墙状态
sudo ufw allow 27017              # 开放 MongoDB 端口
sudo ufw allow 22                 # 开放 SSH 端口
sudo ufw enable                   # 启用防火墙
```

## 6. 常用数据库操作

```bash
# 连接数据库
mongosh "mongodb://user:password@localhost:27017/dbname"

# 数据库操作
show dbs                # 显示所有数据库
use dbname             # 切换数据库
show collections       # 显示所有集合

# 备份和恢复
mongodump --out ./backup                    # 备份数据库
mongorestore --drop ./backup                # 恢复数据库
```

## 注意事项

1. **安全建议**
   - 始终使用强密码
   - 定期更改密码
   - 只开放必要的端口
   - 定期备份数据库

2. **性能优化**
   - 监控数据库大小
   - 定期清理日志
   - 适时优化索引

3. **故障排查**
   - 检查 MongoDB 日志
   - 确认服务状态
   - 验证网络连接
   - 检查磁盘空间

这些命令和配置覆盖了我们在实际使用中最常需要的操作。建议保存这个文档以便随时查阅。
